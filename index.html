Intro to CSS

CSS is a style sheet language used for describing the presentation of a document written in a markup language such as HTML. It is designed to help you create and style your website layouts, fonts, and coloring. It pairs with HTML(Hyper-text Markup Language) while HTML is used to describe content of your website CSS allows you to format and design layouts to help keep people attracted and attentive. It also solved a problem for early web devlopers where HTML does have some style tags built in, you would have to do it for every page, every element your working with. CSS on the other hand allows you to manipulate mutiple pages, elements, selectors, all with one simple sheet completley removed from your html files.

Things to consider

  • HTML and CSS go together, you will need fluent knowledge of both to make beautiful websites.
  • You want some knowledge of the box model.
  • Javascript can also give your websites a nice flare and some animation
  • CSS will only help with Front-End devlopment, it wont help you route anything through servers
  • Some of this documentation will use references from HTML
  • There are references for everything listed at the bottom of the page

Types of CSS

This section will describe the three ways you can use CSS within your websites.

  • Inline CSS

    • This is coded within a tag in your HTML document

    Example:
    <p style = 'color: blue'> example text </p>

    What it does:

    This would change 'example text' to 'example text'

  • Internal CSS

    • This goes generally at the top of your HTML document and will style that page with the selectors you choose

    Example:

    <style>
    .b {
    background-color:blue;
    }

    What it does:

    The code below would change the background color of your b class to blue

  • External CSS

    • This is a seperate document from your HTML with the .css extention that can control mutiple pages unlike Internal CSS.

    You dont need the <style> tag in your css document.

    You do have to link your style sheet in each html document you are writing.

    Example:

    p{
    color:green;
    }

    What it does:

    The code below would change the font color of all p tags that are linked with the .css document

CSS Selectors

Selectors are how CSS recognizes what part of your page its suppose to apply your styles to.

We can narrow it down to five different types of selectors

  • Simple Selectors

    • This is one element selected by name such as an ID or a Class

    IDs and Classes are set in your HTML document and

    IDs are selected with a #

    Classes are selected with a .

    Example:

    .b {
    }
    #b{
    }

    What it does:

    .b will select all tags with the CLASS b so you cant style just those tags with class b

    #b will select all tags with the ID b so you cant style just those tags with ID b

  • Combinator Selectors

    • It is used to select specific relationships between selectors

    Styling all the children of a div, or all the parents of your paragraph tags

    A space between two selectors signifys all decendents within the first selector

    > between two selectors signifys all children to a specific selector

    + between two selctors signifys all adjacent siblings to a specified selector

    ~ between two selectors signifys all siblings to a specified selector

    Example:

    div p{
    }
    div > p {
    }
    div + p {
    }
    div ~ p {
    }

    What it does:

    div p selects all p elements within all divs

    div > p selects all children of p within all divs

    div + p selects all adjacent siblings for the first p elements within all divs

    div ~ p selects all p elements that are siblings of all divs

  • Pseudo-Class Selectors

    • Selects a certian state

    : are used to select pseudo-classes after your selector

    Example:

    button:hover {
    background-color:red;
    }

    What it does:

    When you hover over any buttons on the page, it will change the background of your button to red

  • Pseudo-Element Selectors

    • Selects a certian part of an element.

    Uses :: instead of : to select pseudo elements

    Example:

    p::firstline{
    color:green
    }

    What it does:

    It changes the first line of all p tag's font color to green

  • Attribute Selectors

    • Selects an attribute to be styled

    Example:


    input[type='button']{
    color:black;
    }

    What it does:

    This will make your placeholder text for your buttons black instead of default gray.

Box model

Box model is simple to understand and hard to master

But to be able to align items and understand how things are placed on a website you need to master the box model

Learning it will help you better set your items where they need to go

  • Content

    • Inner most layer of the box model, where all your content and text appears

  • Padding

    • Right beside the content, transparent layer that creates a bit of space around the content

  • Border

    • Goes around the outside of the padding, can be colorful or transparent

  • Margin

    • Final layer also transparent, creates space around the element itself

Flexbox/Grid

You need to understand both models in depth to be able to swap and use both on your pages

This topic is very large, and highly debated on which one is the best and why

This will only go over a few pros and cons of both of them, and ill leave some references for you to do more research

  • Flexbox

    • Pros

    • Easy to grow and shrink elements
    • Better focus on content flow
    • Better browser support
    • Very little code needed to get a layout

    Cons

    • Can't do 2D layouts
    • Preforamce issues with bigger websites
    • Hard to overlap items
    • A bit complicated to deal with whitespace

  • Grid

    • Pros

    • Can make 2D or 1D Layouts
    • Easy to overlap objects
    • Better focus on object positioning
    • Easy to adjust for website responsiveness

    Cons

    • A lot of code must be written to get a grid up
    • Must name lots of elements, and objects
    • Can be complicated to learn
    • The grid will sometimes not provide feedback until you correctly specify all the element names
